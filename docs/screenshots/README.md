# Screenshot Guide

Drop your screenshots into this folder using these exact file names, and they will appear in the main README automatically. Only 8 images are needed.

| File name | What to capture |
|---|---|
| `banner.png` | Wide hero image — the booking map with the app visible (landscape, ~1280x400) |
| `passenger-booking.png` | Booking panel with pickup + destination set and the fare estimate visible |
| `passenger-live-tracking.png` | An accepted ride with the driver marker and route line on the map |
| `assistant-search.png` | AI assistant showing place results with the "Book ride here" button |
| `driver-requests.png` | Driver home with a pending request and the route preview drawn on the map |
| `admin-dashboard.png` | Admin dashboard stats page |
| `admin-live-ops.png` | Live Ops map with active rides and drivers |
| `android-booking.png` | The APK on a phone — booking a ride (portrait) |

Tips:
- Phone screenshots look best at their native portrait resolution.
- For desktop shots, a ~1280px-wide browser window keeps text readable.
- PNG format, no need to resize — GitHub scales them to fit.
